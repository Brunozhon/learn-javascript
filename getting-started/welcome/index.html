We will be learning about lexical structure, the first thing you should learn. It's the set of rules that define how you write a program.

Text

JavaScript is a case-sensitive language. This means that language keywords, function names, variables, and other identifiers must always be typed with a consistent capitalization. The while keyword, for example, can only be typed all in lowercase, that is, while, and not While or WHILE.

JavaScript ignores spaces between tokens in programs. Usually, JavaScript ignores line breaks, but there are a few special cases. Because you can use spaces and newlines freely, you can format and indent your code to be nice and easy to read. For example, which code is easier to read?

           var         x         =           2      +                  3            ;

var     x     =     2     +     3     ;

var x = 2 + 3;

varx=2+3;

The third looks best, of course. The fourth, however, is the worst. The JavaScript interpreter would interpret varx as a variable's name instead of var x. JavaScript also recognizes tabs, ASCII control charcters, and various Unicode whitespace.

Comments

JavaScript supports two types of comments. Any text between // and the end of the line is treated as a comment and ignored by JavaScript. Any text between /* and */ is treated as a comment, too. Comments can't be nested. These are valid comments:

// This is a comment
/* These are */ // comments, too.
/*
 * Comments can span multiple lines, too. The
 * stars at the start of each line aren't
 * required, but they look nice and neat!
*/
/*
See?
*/

Literals

A literal is a data value, like "Hello!" or 3.1415926 appearing directly in a program. They will be explained later.

Identifiers and Reserved Words

Identifiers are used to a name functions, variables, constants, properties, methods, and classes. An identifier can start with a letter, an underscore, or a dollar sign. These are all valid:

x
slithering_snake
theBigCamel
$IHave
